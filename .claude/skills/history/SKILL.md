# 작업 기록 문서 작성 워크플로우

세션에서 수행한 작업을 기록할 때 아래 3개 파일을 업데이트한다.

## 1. 상세 기록 — `docs/phase1/history/YYYY-MM-DD_<topic>.md`

기존 파일 참고: `docs/phase1/history/2026-02-11_rmsnorm.md`

### 필수 섹션 (순서대로)

```markdown
# YYYY-MM-DD: <제목>

## 작업 개요
(한 문단으로 오늘 한 일 요약)

## 배경
(해당 기술이 무엇이고 왜 필요한지)

## 설계 결정
(어떤 접근 방식을 채택했고 왜)
| 선택지 | 설명 | 채택 여부 |

## 구현 상세
(커널 알고리즘, 데이터 흐름, 최적화 전략)

## 생성/수정된 파일
### 신규 생성
| 파일 | 줄 수 | 설명 |
### 기존 파일 수정
| 파일 | 변경 내용 |

## 교육 문서 내용 요약
(.cuh와 .cu에 작성한 한국어 교육 주석 요약)

## 핵심 학습 포인트
(번호 매겨서 3~5개)
```

## 2. 타임라인 업데이트 — `docs/phase1/history/OVERVIEW.md`

### 타임라인 섹션에 항목 추가

```markdown
### YYYY-MM-DD — <제목>

(상세 내용은 `YYYY-MM-DD_<topic>.md` 참조)

- **핵심 구현 1** 설명
- **핵심 구현 2** 설명
- 테스트 N개 + Python 테스트 N개, 전체 N개 통과
```

### 현재 완료 상태 테이블 업데이트

완료된 항목은 `⏳ 미완` → `✅ 완료`로 변경.

### 다음 예정 작업 업데이트

완료된 작업 제거, 새로운 다음 작업 반영.

## 3. 계획 업데이트 — `docs/phase1/PLAN.md`

### 완료된 작업 섹션에 번호 이어서 추가

```markdown
### N. <제목> — `<파일>` (줄수) ✅

설명 + 핵심 함수/클래스 목록.
```
